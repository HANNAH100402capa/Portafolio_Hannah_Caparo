Proyecto 4: Automatización de procesos administrativos

(Caso real)

Diseño de un sistema para automatizar tareas administrativas repetitivas. Análisis del impacto en la eficiencia operativa y en la reducción del trabajo manual.

Volver al Portafolio

Motivación

Este proyecto inició cuando empecé a trabajar en la Sociedad Peruana de Neurología como auxiliar administrativo. Entre mis principales funciones identifiqué que muchas de estas tareas eran repetitivas y demandaban una cantidad considerable de tiempo, para lo cual la institución no contaba con herramientas adecuadas. Este proceso no solo me permitió desarrollar una herramienta funcional, sino que también evidenció la importancia de comprender a profundidad el funcionamiento interno de la institución antes de proponer cualquier solución tecnológica. Haber conocido de primera mano cada tarea, cada flujo y cada necesidad administrativa fue fundamental para diseñar un sistema realmente útil y adaptado a la realidad de la Sociedad Peruana de Neurología. Esta experiencia reafirmó el valor de entender el trabajo en su totalidad como primer paso para lograr una implementación efectiva y sostenible.

Importancia

Este sistema básico, pero funcional, permitió agilizar varias de las tareas diarias de un puesto administrativo. Al evidenciar su eficacia, reconocí que esta solución podía escalarse y convertirse en una herramienta valiosa para automatizar gran parte del flujo administrativo de cualquier sociedad, especialmente en lo relacionado con el envío de certificados y la elaboración de documentos.

Herramientas utilizadas

  • Python: Utilizado para desarrollar y gestionar la lógica principal del sistema.
  • Gmail API: Implementada para permitir el envío seguro y automatizado de correos electrónicos.
  • Excel: Empleado para organizar, depurar y estructurar las bases de datos utilizadas en el proyecto.
  • Git: Utilizado como sistema de control de versiones para registrar cambios en el código, mantener un historial ordenado del desarrollo y permitir restaurar versiones anteriores cuando fue necesario.
  • GitHub: Plataforma empleada para almacenar el repositorio en la nube, facilitar el respaldo del proyecto y organizar de forma segura cada actualización del sistema.

Alcance y Limitaciones

Este proyecto se centra en automatizar las tareas administrativas repetitivas, principalmente relacionadas con la generación y envío de certificados, cartas y la organización de bases de datos. Incluye la automatización del flujo de correos electrónicos mediante Gmail API, la gestión de información en Excel y las inscripciones a los servicios que se ofrecen. No abarca otras áreas administrativas como pagos o gestión financiera.

El sistema depende de la estructura y correcta organización de las bases de datos en Excel; cualquier modificación significativa requiere ajustes en el código. Los envíos automáticos dependen de la autenticación con Gmail API, por lo que cambios en las credenciales pueden interrumpir temporalmente el funcionamiento. Además, existe una restricción diaria de 500 correos por Gmail.

La automatización no reemplaza por completo la supervisión humana, ya que ciertos casos especiales requieren intervención manual. La escalabilidad a otras sociedades o instituciones puede requerir adaptaciones específicas del sistema.

Metodología

1. Identificación de procesos repetitivos: Se analizó el trabajo diario para detectar tareas que consumen tiempo y son susceptibles de automatización.

2. Levantamiento de información: Se documentaron los flujos de trabajo, bases de datos utilizadas y formatos de documento.

3. Desarrollo del sistema: Se implementó el código en Python, integrando Gmail API y Excel para automatizar tareas específicas.

4. Pruebas y ajustes: Se realizaron pruebas con casos reales para garantizar la correcta ejecución de cada función y se corrigieron errores.
